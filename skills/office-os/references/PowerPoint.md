# PowerPoint workflow

Use this reference for .pptx work. Recognize .ppt and .pptm, but request conversion before writing.

## Chunk model

Use slide and shape structure:

1. deck purpose, audience, storyline, and slide inventory;
2. one slide;
3. title, content shape, chart, table, image, notes, and relationship tree;
4. shared layout, master, theme, or repeated component dependency.

Process a 12-slide deck slide by slide when needed, but validate the storyline at deck level and the changed slide at shape level.

## Editing

Preserve theme, masters, layouts, slide sizes, notes, hyperlinks, media relationships, chart data, alt text, and reading order unless the task changes them.

Use the source deck or template as the visual authority. For a new neutral deck, keep one message per slide, a short title that states the point, strong alignment, restrained color, readable type, and charts that answer a question.

## QA routing

Fast:

- package and slide count validation;
- changed slide text, object presence, and overflow checks;
- deck montage for rhythm and outliers;
- full-size inspection of changed or flagged slides.

Enhanced:

- verify masters/layouts/themes, repeated components, chart data, notes, links, animations that affect comprehension, and cross-slide terminology;
- inspect all slides affected by a global change;
- check storyline continuity before and after changed slides.

Full:

- use for explicit request, executive/high-risk delivery, or repeated anomalies;
- inspect every relevant slide full size and verify the full storyline.

Escalate for master/layout/theme changes, chart data changes, global typography, slide-count changes, or low confidence.

## Review findings

Report slide number, shape or region, issue, audience impact, and proposed correction. Distinguish content hierarchy, visual hierarchy, and factual accuracy.
